## What we built BEYOND PRD spec (drift toward more)

Things shipped that weren't in PRD §1-16:

1. **Service lifecycle tooling** — `scripts/service/` (7 PowerShell scripts) + `docs/SERVICE.md`. PRD never specified Windows service install/migrate/teardown. Driven by user request to run as background service with clean teardown for machine migration.
2. **NSSM-based Windows hosting** — Native Windows service via NSSM. PRD assumed Docker. We bypassed Docker per the F-wave audit (NanoClaw v2 spawns containers per-agent-group dynamically).
3. **`pnpm svc` shortcuts** — 10 npm scripts for service ops. Added for ergonomics.
4. **`pnpm ingest:now` helper** — Manual ingestion trigger as TS script (`scripts/ingest-now.ts`). PRD assumed `/ingest` slash command was the only path.
5. **Self-elevating UAC wrappers** — `install-elevated.ps1`, `uninstall-elevated.ps1`. Driven by user not wanting to manually right-click PowerShell.
6. **Path-helper module** — `src/modules/paths.ts` (37 lines). Unifies `~`-expansion across all path env-vars. PRD didn't anticipate Windows-style paths.
7. **Migrate-export/import** — `migrate-export.ps1`, `migrate-import.ps1`. Bundle creds + memory + vault as zip, restore on new machine. Driven by user "this is a test laptop" requirement.
8. **Status skill rewrite** — `groups/pocketclaw/skills/status/SKILL.md` rewrote to use real env-var paths and platform-specific Telegram/WhatsApp formatting.
9. **PRD-distance scorecard notepad** — `.omo/notepads/pocketclaw/prd-distance.md`. Earlier sister doc to this one.
10. **Chat archive (§17.7)** — Already documented in §17, but called out separately because it's the most-recent and most-loaded addition.
