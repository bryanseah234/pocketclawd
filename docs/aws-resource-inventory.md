| Global Cohere Embed Multilingual v3 | `cohere.embed-multilingual-v3` | **Embedding default** (forced 1024-dim) |
| APAC Claude Sonnet 4 | `apac.anthropic.claude-sonnet-4-20250514-v1:0` | (available, not used) |

| APAC Anthropic Claude 3.5 Sonnet | `apac.anthropic.claude-3-5-sonnet-20240620-v1:0` | (available, legacy) |

**Important:** Bedrock requires inference-profile IDs (prefixed `global.` /
`apac.`) instead of bare model IDs for these models. Direct `InvokeModel`
against a bare id returns `ValidationException: ... isn't supported`. Always
use the profile id.

The active model split runs:

| Surface | Secret key | Live value |
|---|---|---|
| Orchestrator (delivery + light classification) | `llm_model_id` | `global.anthropic.claude-sonnet-4-5-20250929-v1:0` |
| Sub-agent (per-user reasoning, RAG) | `llm_subagent_model_id` | `global.anthropic.claude-sonnet-4-5-20250929-v1:0` |
| Embedding pipeline | resolved from region | `cohere.embed-multilingual-v3` (Titan v2 not GA in apse1) |

Containers spawned by `src/cloud/container-manager/lifecycle.ts` get the
sub-agent model id forwarded as `BEDROCK_LLM_MODEL_ID`; the
`BedrockClient` and `BedrockClaude` in `container/sub-agent/src/llm/` honour
it (env > caller arg > `DEFAULT_MODEL_ID`).

## IAM — roles in play

- `nanoclaw-ec2-role` — EC2 instance profile. Inline policies:
  `s3-access`, `dynamodb-access`, `bedrock-invoke`, `secrets-read`,
  `aoss-api-access` (`aoss:APIAccessAll`), `google-secrets-read`, `ssm-managed`.
- `nanoclaw-sub-agent-task-role` — ECS task role. Same shape but scoped to
  the sub-agent's needs (Bedrock invoke, AOSS, DDB read/write per table, S3
  read/write per prefix).
- `nanoclaw-sub-agent-execution-role` — ECS task execution role. ECR pull +
  CloudWatch logs writes only.
- `nanoclaw-github-deploy` — assumed by GitHub Actions via OIDC. ECR
  push + SSM RunShellScript + ECS update-service.
